# Google認証動作確認チェックリスト（Stage2）

対象：Google認証・会員制機能（AUTH-01）

## 前提

- ローカル開発サーバーが起動している（`npm run dev` → `http://localhost:3000`）
- 環境変数が設定されている
  - `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`（Google OAuth）
  - `AUTH_SECRET`
  - `MONGODB_URI`
- Google Cloud Console の OAuth リダイレクト URI に `http://localhost:3000/api/auth/callback/google` が登録されている
- 開発環境（`NODE_ENV !== 'production'`）では、Google 連携なしでも **モックログイン（開発用）** ボタンが表示される

## 確認環境

- **ブラウザ**: Chrome / Edge
- **表示**: PC / タブレット / スマホ（レスポンシブ）
- **認証方式**: Google ログイン（本番相当）またはモックログイン（開発用）

## チェックリスト

### 1. ログインページの表示（`/auth/signin`）

- [ ] 未ログイン状態で `/auth/signin` にアクセスできる
- [ ] コスミック背景（`CosmicBackground`）が表示される
- [ ] カード内に見出し「会員ログイン」が表示される
- [ ] 説明文「Googleアカウントでログインすると、会員専用のダッシュボードとプロフィール編集が利用できます。」が表示される
- [ ] 「Googleでログイン」ボタン（Google アイコン付き・全幅）が表示される
- [ ] 開発環境では「モックログイン（開発用）」ボタンも表示される
- [ ] レイアウト崩れ（はみ出し・重なり・横スクロール）がない
- [ ] 既にログイン済みの場合、`/auth/signin` にアクセスすると `/dashboard` へリダイレクトされる

### 2. Googleログインボタンの動作

- [ ] 「Googleでログイン」ボタンをクリックすると Google の認可画面へ遷移する
- [ ] Google アカウントを選択・許可するとアプリへ戻る
- [ ] 認可をキャンセルした場合、エラーなくログインページ（またはエラー表示）に戻れる
- [ ] 初回ログイン時、MongoDB の `users` コレクションにユーザーが作成される
- [ ] （開発用）「モックログイン（開発用）」ボタンで Google なしにログインできる

### 3. ログイン成功後のリダイレクト

- [ ] ログイン成功後、`/dashboard` にリダイレクトされる
- [ ] 保護ルート（例：`/profile`）から未ログインでアクセスした場合、ログイン後に元のページ（`callbackUrl`）へ戻る
- [ ] リダイレクト後、ブラウザのアドレスバーが正しい URL になっている
- [ ] リダイレクト直後にエラーページや無限ループが発生しない

### 4. ダッシュボードの表示（`/dashboard`）

- [ ] ログイン後 `/dashboard` が表示される
- [ ] 共通ナビ（`MemberLayout`）が表示される
  - [ ] タイトル「会員ダッシュボード」
  - [ ] ナビリンク：「ダッシュボード」「プロフィール」「トップ」
  - [ ] ヘッダーにユーザー名・アバターが表示される
- [ ] 本文に「ようこそ、{名前} さん」が表示される
- [ ] ユーザーのアバター（64×64）とメールアドレスが表示される
- [ ] 会員専用ページの説明文が表示される
- [ ] 「ログアウト」ボタンが表示される
- [ ] コスミック背景・ダークテーマに沿った UI になっている

### 5. プロフィールページの表示（`/profile`）

- [ ] ナビの「プロフィール」リンク、または `/profile` 直接アクセスでページが表示される
- [ ] 見出し「プロフィール」が表示される
- [ ] ユーザーのアバター（72×72）が表示される
- [ ] Google 連携中のメールアドレスが表示される
- [ ] 「Googleアカウントで連携中」の注記が表示される
- [ ] 「表示名」入力欄（`TextField`）に現在の名前が入っている
- [ ] 「変更を保存」ボタンが表示される
- [ ] 共通ナビのユーザー名・アバターがプロフィール情報と一致している

### 6. プロフィール編集の動作

- [ ] 表示名を変更して「変更を保存」をクリックすると保存が完了する
- [ ] 保存成功時に「プロフィールを保存しました」（成功 Alert）が表示される
- [ ] 保存中はボタンが「保存中…」になり、二重送信できない
- [ ] 表示名を空にして保存するとエラーになる（必須バリデーション）
- [ ] 保存後、プロフィールページの表示名が更新される
- [ ] 保存後、ダッシュボードの「ようこそ、{名前} さん」とヘッダーのユーザー名が更新される
- [ ] ページをリロードしても変更後の名前が維持されている（MongoDB 永続化）
- [ ] （API 確認）認証済みで `GET /api/user/profile` がプロフィール JSON を返す
- [ ] （API 確認）`PUT /api/user/profile` に `{ "name": "新しい名前" }` を送ると 200 と更新後プロフィールが返る

### 7. ログアウト機能

- [ ] ダッシュボードの「ログアウト」ボタンをクリックするとログアウトできる
- [ ] ログアウト後、トップページ（`/`）にリダイレクトされる
- [ ] ログアウト後、`/dashboard` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] ログアウト後、`/profile` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] ログアウト後、ブラウザの開発者ツールでセッション Cookie が無効化されている（または再アクセス時に未認証になる）

### 8. 未ログイン時のリダイレクト（保護ルート）

- [ ] 未ログインで `/dashboard` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] 未ログインで `/dashboard/任意のパス` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] 未ログインで `/profile` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] 未ログインで `/profile/任意のパス` にアクセスすると `/auth/signin` へリダイレクトされる
- [ ] 未ログインで `/`（LP）や `/auth/signin` にはアクセスできる（保護対象外）
- [ ] 未ログインで `GET /api/user/profile` を呼ぶと 401 Unauthorized が返る

### 9. セッションの維持

- [ ] ログイン後、ページをリロード（F5）してもログイン状態が維持される
- [ ] ログイン後、別タブで `/dashboard` を開いてもログイン状態が維持される
- [ ] ログイン後、ブラウザを閉じて再度アクセスしてもセッションが維持される（セッション有効期限内）
- [ ] セッション維持中、保護ルートへ直接アクセスしてもログインページへ飛ばされない
- [ ] セッション維持中、ヘッダーのユーザー名・アバターが正しく表示され続ける

## 補足（トラブルシューティング）

| 症状 | 確認ポイント |
|------|-------------|
| Google ログインでエラー | `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`、リダイレクト URI の登録 |
| ログイン後にユーザー情報が取れない | `MONGODB_URI` の接続、MongoDB の起動状態 |
| 保護ルートが開けてしまう | `middleware.ts` の `matcher` 設定、`AUTH_SECRET` の設定 |
| プロフィール保存が失敗する | MongoDB 接続、ネットワークタブで `PUT /api/user/profile` のレスポンス確認 |
| 開発環境でモックログインが出ない | `NODE_ENV` が `production` になっていないか確認 |
