# MVP機能一覧と優先順位

**Stage1のMVP**を「トップページ（LP）のみ」と定義する。技術スタックは **Next.js 15（App Router）**、**MUI** とする。

---

## MVPスコープの境界

| 区分 | 内容 |
|------|------|
| **含む** | 単一URLのランディングページ、静的コンテンツ、CTAボタン（遷移先は後続実装に合わせてプレースホルダ可） |
| **含まない（次Stage候補）** | 診断ロジック、API連携、認証、DB、管理画面、分析タグの本番運用設計の詳細 |

---

## 機能一覧（Stage1 MVP）

| 優先度 | ID | 機能名 | 概要 | 受け入れ条件（要約） |
|:------:|-----|--------|------|----------------------|
| P0 | LP-01 | ヒーローセクション | キャッチコピー、サブコピー、主CTA | ファーストビューで3要素が視認できる。CTAはキーボード・スクリーンリーダーで操作可能。 |
| P0 | LP-02 | サービス特徴（3つ） | 訴求ポイントをカード等で3件表示 | 3件が同じ情報設計（見出し＋短文）で揃っている。 |
| P0 | LP-03 | 診断の流れ（3ステップ） | 利用イメージをステップ表示 | ステップ番号と短い説明が対応している。 |
| P0 | LP-04 | ページ下部CTA | スクロール後の再訴求 | ヒーローCTAと同じ遷移先・同系統のラベル方針。 |
| P0 | LP-05 | フッター | コピーライト、ポリシー・問い合わせ等の導線 | 最低限コピーライト表示。ポリシーURLは `#` や仮リンク可（リリース前に差し替え）。 |
| P1 | LP-06 | レスポンシブレイアウト | 主要ブレークポイントで崩れない | MUIのGrid／Container等でモバイル〜デスクトップで可読。 |
| P1 | LP-07 | アクセシビリティ基礎 | コントラスト、フォーカス、見出し階層 | 主CTAに適切なラベル。見出しは `h1`→`h2` の順序を維持。 |
| P2 | LP-08 | メタデータ | `metadata` による title / description | App Router の `layout.tsx` または `page.tsx` で設定。 |
| P2 | LP-09 | OGP（任意） | SNS共有時の見え方 | 画像・文言は後から差し替え可能な構成でよい。 |

---

## 優先順位の意味

| 優先度 | 意味 |
|--------|------|
| **P0** | Stage1リリースに必須。欠けるとプロダクトとして成立しない。 |
| **P1** | ターゲット（学生・社会人・スマホ利用）に直結。P0直後に実装。 |
| **P2** | リリース品質・拡散に効く。リソースがあれば同一Stage内で対応。 |

---

## 技術実装メモ（要件レベル）

| 項目 | 方針 |
|------|------|
| ルーティング | App Router のルート（例：`app/page.tsx`）にLPを集約 |
| UI | MUI（`@mui/material`）でヒーロー、特徴、ステップ、CTA、フッターを構成 |
| スタイル | 既存テーマとTypographyの統一。独自ブランドカラーはテーマに集約すると改修が容易 |

---

## 次Stage（参考：優先度ざっくり）

| 優先度 | 候補 | 備考 |
|:------:|------|------|
| 高 | 診断設問UI（5問）、回答送信、結果表示 | バックエンド・プロンプト設計と一体 |
| 高 | CTAの実遷移・完了率計測 | アナリティクス要件を別紙化 |
| 中 | 会員化・結果保存 | LP単体からの拡張 |
| 低 | A/Bテスト用ヒーロー差し替え | トラフィック後 |
